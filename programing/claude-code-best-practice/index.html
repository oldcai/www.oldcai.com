Claude Code 最佳实践

oldcai
2025-08-11
Claude Code, Vibe Coding

CLAUDE.md 文件

Claude Code像金鱼一样，是没有长期记忆的。他就只靠这一个文件来理解这个项目。
作为开发者，我们要维护CLAUDE.md文件，让 Claude Code 每次运行都更容易理解这个项目。

权限管理

我在做原型的时候，一般会让它拥有所有的权限，在一个安全的沙盒环境里面去运行。但是在做严肃的项目的时候，可能就要让它每次进行写操作和执行 bash 命令的时候都需要我去允许。

在使用 Claude Code 的时候，用快捷键 Shift + Tab 可以切换权限模式。有时候你可以全部允许，有时候你用 plan 模式（完全不允许文件写入）。

怎样修改 Bug？

  • 你不应该这么说：我需要你修复这个 bug（描述bug）

  • 你应该这么说：我有一个这样的 bug（描述bug），你能不能在代码里面搜索一下，找到什么导致的这个 bug，然后告诉我复他计划。

这样可以节省很多时间。因为你可以去验证 Claude 的计划，而不是让他胡乱改代码，让整个 commit 不可控。
老菜在这个上面吃过亏，甚至最后只能靠自己手动去修改代码。但是这其实可以避免。可以在理解整个项目代码实现的原理和结构的前提下，通过上面的和 Claude Code 沟通的方法来实现。

看到事情不对怎么办？

当我们看到 Claude Code 正在朝着奇怪的方向修改代码的时候，我们可以按一下 ESC 键，让它停止当前的计划，然后我们告诉他接下来应该怎么做。

TDD

让 Claude Code 每次只修少量修改代码，不断迭代，并且通过所有测试。

版本管理

让 Claude Code 时不时就就提交一下代码，这样你在出错的时候可以方便回滚。

直接粘贴图片

Claude Code 是多模态的，我们可以随时把我们的 mockup 或者是屏幕截图发给他。快捷键是 Ctrl + v。但是要注意，它只能在本机上面这样粘贴图片，如果在远程服务器上面就不太行了。要先上传图片，然后再@这张图片。

Claude Code 高级技巧

ESC 键的使用

  • 按一次: 中断当前的执行。
  • 按两次: 回溯之前的对话。虽然说 Claude Code 不能将已经修改的代码回缩到之前的 checkpoint快照，但是它可以将我们当前的会话恢复到之前的历史记录情景中。

使用魔法命令

  • think hard
  • ultrathink

现在 Claude Code 在两次工具调用之间也可以思考。所以我们在找 bug 或者是需要分析的时候，加上一个魔法命令，会极大的提高成功率。

多个 agent 之间怎样交流？

他们可以共享一个 .md 文件。 比如告诉 agents A 将某一个工作的描述写到这个文件 ticket.md 中，然后告诉 agent B 从 @ticket.md 文件中读取他的工作描述。
